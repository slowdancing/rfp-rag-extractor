## LLM 비교

### 정성 인사이트
- **Qwen2.5-7B**: 한국어 답변 뒤에 **중국어 번역이 혼입**(중국 모델 특성).
  프롬프트로 "한국어만" 강제해 완화되나 근본 제약 존재.
- → 한국어 특화 **EXAONE** 채택 방향.

### 정량 비교 — 경량 양자화 모델 (VM, Ollama)
VM 메모리 안전을 위해 **경량 양자화 모델**을 한 개씩 실행(결과는 `compare_llms.json` 원장 누적,
표는 `compare_llms.md` 자동 생성). 평가셋 `eval_set.draft.jsonl` 표본 30건, 임베딩 bge-m3 고정.

| LLM (경량) | 파라미터 | 정확도 | 정답/전체 |
|------|---:|------:|------:|
| **exaone3.5:2.4b** | 2.4B | **0.467** | 14/30 |
| gemma2:2b | 2.0B | 0.367 | 11/30 |
| qwen2.5:3b | 3.0B | 0.333 | 10/30 |

- **한국어 특화 EXAONE이 가장 작은 파라미터(2.4B)로도 1위**, 더 큰 qwen2.5:3B가 최하위.
  → 한국어 RFP에선 **파라미터 수보다 한국어 특화가 중요** (임베딩 bge-m3 > OpenAI와 동일한 결론).
- **경량·양자화 환경에서도 EXAONE 채택이 타당**함을 뒷받침.
- 주의: 이 평가셋은 날짜 표기 불일치로 deadline 유형이 0점 처리되는 채점 한계가 있어
  절대 정확도는 낮게 나온다(`eval_generation.md` 참조). **모델 간 상대 순위**가 신호.
